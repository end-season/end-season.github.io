# 可视化UI设计



## 伙伴关系与Tab顺序

* 伙伴关系是指界面上一个标签和一个输入焦点的组件相关联。 请看以下示例：

  ![](/img_post/Qt6_2.2.png)

  在编辑伙伴模式下，鼠标左键点击标签，并将其托向一个输入焦点的组件，这样就将标签与编辑框关联起来，也可在标签后加上（&N）来设置快捷键，括号中N可换为任意字母。

* Tab顺序是指在程序运行时，按下键盘上的`Tab`键时输入焦点的移动顺序。

* 进入Tab顺序编辑状态后，界面上会显示具有Tab顺序的组件的Tab顺序编号，以此按希望的顺序点击组件，就可以重排Tab顺序。没有输入焦点的组件是没有Tab顺序的，例如标签。

## 信号与槽

* 信号（signal）是在特定情况下被发射的通知，例如`QPushButton`较常见的信号就是`clicked`信号。GUI程序设计的主要工作就是对界面上各组件的信号进行响应，只需要知道什么情况下发射哪些信号，合理地去相映和处理这些信号就可以了。
* 槽（slot）是对信号进行响应的函数。槽就是函数，所以也被称为槽函数。槽函数与一般的c++函数一样，可以具有任何参数，也可以被直接调用。槽函数与一般函数的不同是：槽函数可以与信号关联，当信号被发射时，关联的槽函数被自动运行。
* 信号与槽函数关联是用`QObject::connect()`实现的，具体参数可以查阅相关文档。
* 关于信号和槽的使用，有以下几点需要注意：
  1. 一个信号可以连接多个槽。
  2. 多个信号可以连接同一个槽函数。
  3. 一个信号可以连接另一个信号。
  4. 严格的情况下，信号和槽的参数个数和类型需要一致，至少信号的参数不能小于槽的参数。如果参数不匹配，会出现编译错误或运行错误。
  5. 在使用信号和槽的类中，必须在类的定义中插入宏`Q_OBJECT`。
  6. 当一个信号被发射时，与其关联的槽函数通常被立即执行，就像正常的函数调用一样。只有当信号关联的所有槽函数执行完毕后，才运行发射信号处后面的代码。

## 信号与槽的使用

* `QDialog`是对话框类，它有以下几个公有的槽函数：

  * `accept()`，功能是关闭对话框，表示肯定的选择。

  * `reject()`，功能是关闭对话框，表示否定的选择。

  * `close()`，功能是关闭对话框。

    > 虽然`reject()`与`close()`都是关闭对话框，但它们也有以下差别：
    >
    > * `reject()`会使`exec()`返回`QDialog::Rejected`，`close()`虽然默认也是这样，但可以通过重写改变。
    > * `reject()`会触发`rejected`信号，`close()`会触发`closeEvent`信号。

* 信号和槽使用示例：

  ![](/img_post/Qt6_2.1.png)

  我们需要在选中斜体的时候将编辑框中的字体带上下划线：

  * 右键"斜体"->转到槽

  * 在自动生成槽函数中写入以下代码：

    ```c++
    void Dialog::on_checkBox_clicked(bool checked)
    {
        auto font = ui->plainTextEdit->font();
        font.setUnderline(checked);
        ui->plainTextEdit->setFont(font);
    }
    ```

    运行程序，便可以看到想要的效果。

  > 通过UI界面设置的信号与槽的关联，也就是`connect`函数会生成在`ui_dialog.h`文件中。



## Qt项目构建过程基本原理

* 一个Qt项目除项目配置文件外，还由三类文件组成：

  * Qt c++编写的头文件和源程序文件。
  * 窗口资源文件。
  * 资源文件。

  这三类文件会被分别编译为标准c++语言的程序文件，然后再被标准c++编译器编译成可执行文件或者库。

  ![](/img_post/Qt6_2.3.png)

* 元对象系统和MOC

  Qt对标准c++语言进行了扩展，引入了元对象系统(meta_object system, MOS)，所有从`QObject`继承的类都可以利用元对象系统提供的功能。元对象系统支持属性、信号与槽、动态类型转换等特性。

  我们在Qt Creater中编写程序时使用的c++语言，实际上使经过Qt扩展的c++语言。例如，Dialog类中的类定义中插入一个宏`Q_OBJECT`，这是使用信号与槽机制的类必须插入的一个宏。Dialog类中的`private slots`部分用于定义私有槽，这是c++语言没有的特性。

  Qt提供了元对象编译器（MOC）。在构建项目时，项目中的头文件会先被MOC预编译。

* UI文件和UIC

  在构建项目时，可视化设计的窗口UI文件会被用户界面编译器（UIC）转换为一个c++源程序文件。UIC编译生成的文件与UI文件在同一个文件夹里。

* 资源文件和RCC

  Qt项目中的资源文件会被资源编译器（RCC）转换为c++程序文件。

使用MOC、UIC和RCC编译各原始文件的过程被称为预编译过程，预编译之后生成的使标准c++语言的程序文件，它们被标准c++编译器编译和链接，最终生成可执行文件。
